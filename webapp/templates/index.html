Quick Start

  1. Choose Local folder or a Public Git URL.
  2. Select the tools to build a tree, concatenate text files, or both.
  3. Review excludes so noisy folders like .git and venv stay out.
  4. Click Run Tools, then inspect the Structure, Concatenated files, and Logs tabs.

Local folders are processed in this browser tab. They are not uploaded.

Public GitHub repos are fetched via the jsDelivr CDN (no proxy needed); other hosts use a client-side git clone. For a non-default branch, paste the …/tree/<branch> URL. Larger repos are slower than picking a local folder.
Advanced: CORS proxy (non-GitHub hosts) GitHub repos don't use this — they're fetched through the jsDelivr CDN, which serves CORS headers directly. For other hosts (GitLab, Bitbucket, …) browsers can't speak the raw git protocol, so the clone is relayed through a CORS proxy. The default is a public proxy run by the isomorphic-git project; the repository is public and no credentials are ever sent. If it's down or rate-limited (clones fail with a network error), point this at your own proxy, or clear it if the host serves CORS headers directly.

Project Tools Runner Help

Web UI, desktop UI, and local MCP server

A privacy-first toolkit for exploring a codebase: build an ASCII-style project tree, concatenate readable files, and generate logs for review or AI-assisted workflows.

Web UI

This browser app analyzes a local folder or public Git URL, then shows the Structure, Concatenated files, and Logs tabs. Local folders are processed in your browser and are not uploaded.

Desktop UI

The Tkinter desktop app provides the same core tools for local workflows where you prefer a native window.

uv run python tool_runner_ui.py

MCP Server

The local MCP server exposes read-only project tools to MCP-capable AI clients with allowed-root and secret-file protections.

uv sync --extra mcp uv run python mcp_server.py --root /path/to/project

git clone https://github.com/sidbetatester/AI-Dev-Kit.git cd AI-Dev-Kit uv sync uv run python webapp/app.py
